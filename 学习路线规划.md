## 1. 项目概述

### 1.1 这个项目做了什么？

用 Python 对一家英国电商平台一年的交易数据进行分析，包含 **5 个核心分析模块**：

| 模块 | 解决的问题 | 技术点 |
|------|-----------|--------|
| 描述性分析 Dashboard | 整体业务状况如何？ | pandas 聚合、matplotlib 多子图 |
| 国家/市场分析 | 哪个市场贡献最大？ | groupby 聚合、柱状图 |
| 产品分析 | 哪些产品卖得最好？ | 排序、Top-N 分析 |
| RFM 客户分层 | 哪些客户最有价值？ | Recency/Frequency/Monetary 打分 |
| 购物篮分析 | 哪些商品经常一起买？ | 关联规则（Support/Confidence/Lift） |
| 同期群分析（Cohort） | 客户留存在逐月改善还是恶化？ | 留存率矩阵、热力图 |

### 1.2 技术栈

```
Python 3.x
├── pandas       → 数据处理（DataFrame 操作、groupby、pivot）
├── numpy        → 数值计算
├── matplotlib   → 底层绑图
├── seaborn      → 统计可视化（热力图）
└── scikit-learn → 机器学习（本项目中用于数据标准化，实际未深度使用）
```

### 1.3 数据来源

**UCI Machine Learning Repository — Online Retail Dataset**
- 包含 2010/12/01 到 2011/12/09 英国电商平台的 54 万笔交易
- 字段：InvoiceNo（发票号）、StockCode（商品编号）、Description（商品描述）、Quantity（数量）、InvoiceDate（日期）、UnitPrice（单价）、CustomerID（客户ID）、Country（国家）

### 3.4 关键代码注解

#### 数据清洗

```python
# 查看每列有多少空值
df.isnull().sum()
# CustomerID 有 135,080 个空值 — 这是因为有些交易是匿名购买
# Description 有 1,454 个空值 — 商品描述缺失

# 删掉描述为空的行（只有1454行，可以接受）
df = df.dropna(subset=['Description'])
# 删掉 CustomerID 为空的行（13.5万行，占总数据25%！这是一个trade-off）
df = df.dropna(subset=['CustomerID'])

# 删除重复行
df = df.drop_duplicates()

# 过滤掉退货和异常（Quantity ≤ 0 是退货/冲账，UnitPrice ≤ 0 是脏数据）
df = df[(df['Quantity'] > 0) & (df['UnitPrice'] > 0)]
# 这一步过滤掉了 8,872 + 40 = 8,912 行

# 把日期字符串转成 datetime 类型（这样才能做时间运算）
df['InvoiceDate'] = pd.to_datetime(df['InvoiceDate'])
```

#### RFM 打分

```python
# Recency = 距离分析基准日有多少天没买了（越小越好）
# 基准日 = 数据中最后一天 + 1
analysis_date = df['InvoiceDate'].max() + timedelta(days=1)

rfm_data = df.groupby('CustomerID').agg({
    'InvoiceDate': lambda x: (analysis_date - x.max()).days,  # R：最近一次购买距今天数
    'InvoiceNo': 'nunique',                                    # F：购买了多少次
    'TotalAmount': 'sum'                                       # M：总共花了多少钱
})

# pd.qcut 按分位数分成5档，1分最差，5分最好
# 注意 Recency 是反的：天数越少越好，所以用 6 - score
rfm_data['R_Score'] = 6 - pd.qcut(rfm_data['Recency'], q=5, labels=[1,2,3,4,5])
rfm_data['F_Score'] = pd.qcut(rfm_data['Frequency'], q=5, labels=[1,2,3,4,5])
rfm_data['M_Score'] = pd.qcut(rfm_data['Monetary'], q=5, labels=[1,2,3,4,5])

# 拼成三位数，555 = 最近买 + 经常买 + 花得多 = 冠军客户
rfm_data['RFM_Score'] = rfm_data['R_Score'].astype(str) + rfm_data['F_Score'].astype(str) + rfm_data['M_Score'].astype(str)
```

#### 同期群分析（Cohort）

```python
# 1. 每个客户的首次购买月份 = 他属于哪个同期群
df_cohort = df.groupby('CustomerID')['OrderMonth'].min().reset_index()
df_cohort.columns = ['CustomerID', 'CohortMonth']

# 2. 合并回原数据
df = df.merge(df_cohort, on='CustomerID')

# 3. 计算第几期（首次购买后第几个月）
df['PeriodNumber'] = (df['OrderMonth'] - df['CohortMonth']).dt.days // 30

# 4. 透视：行=同期群月份，列=第几期，值=留存客户数
cohort_data = df.groupby(['CohortMonth', 'PeriodNumber'])['CustomerID'].nunique().unstack()

# 5. 留存率 = 每期人数 / 同期群初始人数
retention = cohort_data.divide(cohort_data.iloc[:, 0], axis=0)
```

创建 `app.py`：

```python
import streamlit as st
import pandas as pd
import plotly.express as px
import plotly.graph_objects as go

st.set_page_config(page_title="电商数据分析 Dashboard", layout="wide")
st.title("🛒 电商数据分析平台")

# 加载数据（缓存，不要每次交互都重新加载）
@st.cache_data
def load_data():
    df = pd.read_csv('data/data.csv', encoding='ISO-8859-1')
    # 数据清洗逻辑（和 Notebook 一致）
    df = df.dropna(subset=['CustomerID', 'Description'])
    df = df[(df['Quantity'] > 0) & (df['UnitPrice'] > 0)]
    df['InvoiceDate'] = pd.to_datetime(df['InvoiceDate'])
    df['TotalAmount'] = df['Quantity'] * df['UnitPrice']
    return df

df = load_data()

# 侧边栏 — 筛选器
st.sidebar.header("筛选条件")
countries = st.sidebar.multiselect("选择国家", df['Country'].unique(), default=['United Kingdom'])
date_range = st.sidebar.date_input("日期范围", [])

# 主区域 — KPI 指标卡片
col1, col2, col3, col4 = st.columns(4)
col1.metric("总营收", f"£{df['TotalAmount'].sum():,.0f}")
col2.metric("客户数", f"{df['CustomerID'].nunique():,}")
col3.metric("订单数", f"{df['InvoiceNo'].nunique():,}")
col4.metric("客单价", f"£{df['TotalAmount'].sum() / df['InvoiceNo'].nunique():.2f}")

# 图表区域
tab1, tab2, tab3 = st.tabs(["销售趋势", "客户分析", "产品分析"])

with tab1:
    st.subheader("月度销售额趋势")
    monthly = df.groupby(df['InvoiceDate'].dt.to_period('M'))['TotalAmount'].sum().reset_index()
    monthly['InvoiceDate'] = monthly['InvoiceDate'].astype(str)
    fig = px.area(monthly, x='InvoiceDate', y='TotalAmount', title="月度销售额")
    st.plotly_chart(fig, use_container_width=True)

with tab2:
    st.subheader("RFM 客户分层")
    # ... 你的 RFM 代码
    pass

with tab3:
    st.subheader("热销产品 Top 10")
    # ... 你的产品分析代码
    pass
```

## 10. 面试可能问到的问题

### 10.1 数据清洗相关

**Q1：CustomerID 有 13.5 万行缺失，为什么直接删了而不是填充？**
> CustomerID 是客户标识，不存在"猜一个"的可能。缺失的 CustomerID 可能是匿名购买/游客，无法追踪客户行为，删掉是合理的。但这意味着我们丢失了 25% 的交易数据，分析结论对"有账号的客户"有效，不代表全部交易。如果业务关心全量，可以用 Session ID 或 Cookie 替代 CustomerID。

**Q2：为什么过滤 Quantity ≤ 0 和 UnitPrice ≤ 0？**
> Quantity ≤ 0 是退货/冲账交易，UnitPrice ≤ 0 是数据异常或赠品。如果保留这些数据，总营收会被拉低，产品排名和客户价值计算都会失真。正确的做法是先分离正负交易，正向做分析，负向做退货分析。

**Q3：数据清洗后从 54 万降到 39 万行，这个损失能接受吗？**
> 看分析目标。如果目标是客户行为分析，丢失匿名用户是可以接受的；如果目标是全量交易复盘，就需要考虑要不要保留 CustomerID 缺失的行做整体趋势分析。这是一个 trade-off，需要在分析报告中说明。

### 10.2 RFM 相关

**Q4：RFM 打分的分位数（qcut）是什么？为什么用 5 档？**
> qcut 把数据按分位数切成等量的桶。5 档是业界惯例，来源于五分位数。q=5 意味着每个客户在 R/F/M 三个维度上分别被分成 5 个等级（1-5），组合起来是 125 种可能，然后合并成 10 个业务可理解的群体。

**Q5：Recency 为什么是 6 - score？**
> 因为 Recency 值越小越好（最近刚买过 = 好客户），但 qcut 默认 1=最小值（即最好的）。为了统一方向（5 分 = 最好），用 6 减一下把 Recency 翻过来。

**Q6：RFM 分段的规则怎么定的？**
> 原始 Notebook 的作者参考了 RFM 分析的经典文献（Hughes, 1994 等），把 125 种 RFM Score 组合映射到 10 个业务群体。这些规则不是绝对的——不同业务、不同数据分布下需要调整。

**Q7：你做的 RFM 分段有什么局限？**
> 1）使用的是静态基准日（数据最后一天），没有考虑客户生命周期动态变化；2）分段阈值是固定的，没有根据实际业务迭代；3）只用了交易数据，没有产品偏好、浏览行为等维度。

### 10.3 购物篮分析相关

**Q8：Support、Confidence、Lift 分别是什么意思？**
> - **Support（支持度）**：A和B同时出现的概率。Support = 0.05 表示"5%的交易同时买了A和B"
> - **Confidence（置信度）**：买了A的人中有多少也买了B。Confidence = 0.3 表示"买了A的人中 30% 也买了B"
> - **Lift（提升度）**：Confidence / B的自然购买概率。Lift = 2 表示"买A把你买B的概率提高了2倍"
> - Lift > 1 才有意义（正关联），Lift < 1 是替代品（负关联）

**Q9：你设置的 min_support = 0.01，太大了怎么办？太小了怎么办？**
> 太大 → 发现不了关联 → 降低阈值。太小 → 太多噪音规则 → 提高阈值或按 Lift 排序取 Top N。阈值没有绝对值，取决于数据规模和业务需求。

**Q10：为什么不直接用 `mlxtend` 或 `apyori` 库的现成实现？**
> 手动实现是为了理解算法原理。生产环境肯定用现成库，比如 `mlxtend.frequent_patterns.apriori` + `association_rules` 比自己写的快几百倍。

### 10.4 同期群分析相关

**Q11：Cohort Analysis 的业务价值是什么？**
> 不是看"这个月赚了多少"，而是看"这个月拉来的客户，之后几个月还在不在"。帮助判断获客质量是否在下降、运营策略是否有效、哪个渠道的客户质量更高。

**Q12：留存率怎么定义的？**
> 第 N 期留存率 = 第 0 期（首次购买）的客户中，第 N 期仍然有购买的人数 / 第 0 期总人数。比如 2011-01 Cohort 有 400 人，2011-02 有 120 人回购，第 1 期留存率 = 30%。

**Q13：你做 Cohort 的时候发现留存下降了，可能是什么原因？**
> 1）获客渠道质量下降（比如开始投便宜广告拉来的低质量用户）。2）产品体验变差。3）季节性因素。4）竞品抢走了用户。需要有渠道/产品变更数据才能判断。

### 10.5 通用问题

**Q14：这个项目最大的难点是什么？你最自豪的部分是什么？**
